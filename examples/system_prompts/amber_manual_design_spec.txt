# The Amber Manual — documentation style spec

You are typesetting technical documentation as ONE self-contained static HTML file in
the **Amber Manual** style: a technical field manual from computing's golden age —
somewhere between a DEC terminal handbook and an early O'Reilly. A calm, light reading
page into which dark "plates" (code, terminal output) are set like printed figures.
Follow every spec below exactly; where the spec is silent, choose quietly.

## 1. Design identity (non-negotiable)

- Light page, dark plates. The page is paper; code and terminal output appear as dark
  panels set INTO it. Never a dark page overall.
- The glow accent is **amber phosphor**, never green. Green-on-black is forbidden as an
  identity; it may appear only inside syntax-highlighted strings.
- Maroon is the editorial accent (chapter numbers, links, part labels) — the old
  "digital"-logo red, muted.
- One signature element per document (see §6 hero). Spend boldness there; keep
  everything else disciplined.
- Structure encodes information: numbering, dividers, and labels must state something
  true about the content (real artifact strings, real line counts, real listing
  numbers), never decoration.

## 2. Color tokens (use as CSS variables, verbatim)

```css
:root {
  --paper:      #F4F5F1;  /* near-neutral; reads cream on warm displays.
                             For a deliberately warmer cream use #F5F2E8 */
  --paper-deep: #EAECE6;  /* sidebar, callout fills, diagram mats */
  --chip:       #E6E8E1;  /* inline-code chips, hover fills */
  --ink:        #232A31;  /* body text, blue-black */
  --ink-soft:   #525C66;  /* secondary text, captions */
  --ink-faint:  #8B949D;  /* metadata, footers */
  --maroon:     #8C3A32;  /* chapter numbers, links, part labels, warn accents */
  --maroon-soft:#A85B52;  /* link hover */
  --amber:      #C77F14;  /* accent on light backgrounds */
  --amber-hi:   #FFC163;  /* accent on dark plates: filenames, cursor, active marks */
  --plate:      #14171D;  /* dark panel background (blue-black, not pure black) */
  --plate-2:    #1B1F27;  /* panel titlebars */
  --plate-edge: #2C323D;  /* panel borders, 1px */
  --plate-text: #D7DCE3;  /* default text on plates */
  --plate-dim:  #7C8693;  /* secondary text on plates */
  --hairline:   #D4D7CF;  /* rules on paper */
}
```

Selection: amber-hi background, plate-colored text. Focus-visible: 2px amber outline,
2px offset.

## 3. Typography

Entire document in the IBM Plex family (Google Fonts CDN, with fallbacks):

- Display: **IBM Plex Sans Condensed** 600–700, UPPERCASE for the title and part
  titles; mixed case for h2/h3. Fallback: "Arial Narrow", Arial.
- Body: **IBM Plex Serif** 400, 17px, line-height 1.68. Italic for emphasis and
  figcaptions. Fallback: Georgia.
- Utility/code: **IBM Plex Mono** for all code, labels, tables, kbd, TOC, eyebrows.
  Fallback: Consolas, monospace.

Scale: hero title clamp(3rem, 6.5vw, 4.6rem)/0.98 · part title clamp(2.1rem, 4vw,
3rem) · h2 1.85rem with the chapter number beside it in 1.05rem mono maroon · h3
1.18rem Sans Condensed 600 · body 17px · plate code .8rem/1.58 · captions .88rem
italic · labels/eyebrows .66–.78rem mono, UPPERCASE, letter-spacing .12–.24em.

Lead paragraph after each h2: 1.09rem, full ink. Inline code: .86em mono 500 on a
--chip rounded 4px. kbd: keycap look — light gradient fill, 1px hairline border,
2px bottom border, 4px radius.

## 4. Layout

- Fixed left sidebar TOC, 16.5rem, --paper-deep, 1px hairline right border, mono.
  Entries: part rows in .68rem uppercase maroon; chapter rows .78rem ink-soft with the
  chapter number right-aligned in a faint column. Active chapter: 2px amber left
  border, --chip fill, amber number. Footer line in .66rem ink-faint (version, stack).
- Main column: content max-width 46rem, centered in the remaining space; the hero may
  break out to 54rem. Generous bottom padding (6rem).
- Below 1080px: sidebar slides off-canvas (transform, .22s), toggled by a fixed
  "contents" button — dark plate fill, amber-hi text, mono uppercase.
- Anchor targets get scroll-margin-top so headings clear the viewport edge.

## 5. Recurring components (markup patterns)

**Part divider** — opens each part:
```html
<header class="part" id="part-x">
  <p class="part-cap"><code>ARTIFACT STRING</code></p>   <!-- see rule below -->
  <p class="part-label">Part II</p>
  <h1 class="part-title">THE ENGINE</h1>
</header>
```
part-cap: .72rem mono ink-faint with a hairline bottom border; part-label: maroon
uppercase letterspaced; part-title: huge condensed uppercase. The ARTIFACT STRING must
be a REAL string from the subject's domain that foreshadows the part (for terminal
docs: a terminfo capability like `cup=\E[%i%p1%d;%p2%dH`; for an API: a real request
line; for ML infra: a real config key or launch flag; for networking: a real packet
field). If one part genuinely changes domains, its artifact may break the pattern —
and say so in a clause.

**Chapter heading**:
```html
<section class="chapter" id="ch-x">
  <h2><span class="ch-no">9</span>The Keyboard Is Hostile</h2>
```

**Code plate** — every code block:
```html
<figure class="plate">
  <div class="plate-bar"><span class="plate-name">file.py · ClassName</span>
    <span class="plate-meta">42 lines</span>
    <button class="copy" type="button">copy</button></div>
  <pre class="plate-code"><code class="language-python">…escaped…</code></pre>
  <figcaption><b>Listing 9.3</b>&ensp;One full sentence saying why this code matters.</figcaption>
</figure>
```
Bar: --plate-2, amber-hi name, dim meta with the TRUE line count, ghost copy button
(mono uppercase .66rem, plate-edge border; hover amber; "copied" state fills amber).
Plate: --plate, 1px plate-edge border, 7px radius (bar and pre share the rounding),
overflow-x auto. Caption: italic serif, with the "Listing N.M" tag in bold mono
maroon uppercase. Listings number per chapter and never lie.

**Terminal mock** (hand-set screenshots): a .term window — slim titlebar with one
amber dot and a title — containing `<pre class="screen">` at .8rem/1.5 mono on
--plate, with a faint cell-grid overlay:
`repeating-linear-gradient(to bottom, transparent 0, transparent calc(1.5em - 1px),
rgba(255,255,255,.025) calc(1.5em - 1px), rgba(255,255,255,.025) 1.5em)`.
A blinking block cursor `<span class="cursor">█</span>` in amber-hi,
`steps(1) 1.1s` blink, disabled under prefers-reduced-motion.

**Callouts**:
```html
<div class="callout"><span class="callout-label">At the keyboard</span><p>…</p></div>
```
--paper-deep fill, 3px amber left border, label in mono caps amber. Variants:
`.warn` (maroon border+label, for gotchas) and `.lore` (ink-soft label on --chip,
for history). Use labels like FIELD NOTE / GOTCHA / TERMINAL LORE / AT THE KEYBOARD —
short, uppercase, consistent.

**Tables** (reference/appendix only): mono .82rem, 2px ink rule under the header row,
hairline row rules, no vertical borders, italic serif caption above.

## 6. The hero (signature element)

Open with: a mono uppercase maroon eyebrow → the title huge in condensed uppercase →
a serif standfirst (max-width 40rem, ink-soft, with one or two strong/em phrases) →
the signature: a hand-set terminal mock (per §5) showing the SUBJECT actually
working, wide (54rem breakout) → a short italic note beneath admitting the screenshot
is hand-set HTML, "a lie in the way all good diagrams are." If the subject has no
terminal surface, the signature becomes the subject's most characteristic artifact
rendered as a plate (a config, a trace, a session). Exactly one signature; nothing
else on the page competes with it.

## 7. Code highlighting & diagrams

Syntax theme (highlight.js 11.x from cdnjs, custom colors — no stock theme):
keyword/literal #CF9BD8 · string #9CCB87 · comment #707B89 italic · number #E5A455 ·
function title #7FA8D9 · class title #6FC3DF · built-in/type #84C5B8 · meta #6FC3DF ·
attr/name #E9C46A. Plate text otherwise #D7DCE3.

Inline SVG figures: 700-wide viewBox, mono text 11–13px, fills/strokes drawn from §2
(paper boxes with ink borders, paper-deep with maroon for middle layers, plates with
amber-hi for terminal-side boxes), arrows in ink-soft with small triangular markers
(unique marker ids per figure), caption as `<figcaption><b>Figure N.M</b>&ensp;…`.
Mermaid (10.x, cdnjs) themed via themeVariables to the same hexes, wrapped in a
--paper-deep mat with hairline border.

## 8. Behavior (vanilla JS, embedded)

Scrollspy: IntersectionObserver on chapters, rootMargin "-12% 0px -78% 0px", toggles
the active TOC row. Copy buttons: clipboard API with execCommand fallback, 1.6s
"copied" state. Mobile TOC toggle with aria-expanded. No other animation beyond the
cursor blink and hover states.

## 9. Quality floor

Single .html file; CSS and JS embedded; CDN allowed only for IBM Plex (Google Fonts),
highlight.js, mermaid — always with working fallback stacks. Semantic HTML
(header/nav/main/section/figure). Print stylesheet: hide TOC and copy buttons, plates
go white with gray borders, avoid breaking inside plates. prefers-reduced-motion
respected. All ids stable and linkable.

## 10. Voice

Body is flowing book prose — no bullet lists in chapters (tables and figure content
excepted; appendices may use tables). Captions are full sentences that add
information, never restate the title. Labels are sentence-case in UI, uppercase only
in the mono label system. Be specific over clever; numbers must be true (line counts,
listing numbers, version strings).

## 11. Forbidden

Dark page backgrounds · green-on-black identity · gradients (except the keycap and
cell-grid) · rounded-card grids · emoji in headings · stock highlight.js themes ·
decorative numbering that doesn't count anything · drop shadows beyond the single
plate shadow `0 1px 2px rgba(20,23,29,.18), 0 14px 38px -18px rgba(20,23,29,.45)` ·
fabricated screenshots presented as real.
